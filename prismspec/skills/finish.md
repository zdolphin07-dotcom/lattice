# Skill: finish — PrismSpec Closeout

**Triggers**: `/finish`, finish, close spec, summarize delivery

## Capability

Close the PrismSpec run by writing `summary.md` and preserving only durable knowledge.

## Workflow

1. Read `spec.md`, `plan.md`, and verification evidence.
2. Inspect relevant git status and changed files.
3. Write `summary.md` next to the spec.
4. Include:
   - what changed;
   - verification evidence;
   - deferred work;
   - reusable lessons;
   - known gaps in the workflow or project.
5. Update spec status when possible.

## Exit Criteria

- `summary.md` exists.
- Verification result is explicit.
- Follow-up work is concrete and scoped.
